    **Step 1: Add DayPlan to schedule_models.py** (per MDS-05)

    Add `from datetime import date` import (datetime already imported). Add DayPlan dataclass after the ChargeSchedule class:

    ```python
    @dataclass
    class DayPlan:
        """Per-day charge plan within a multi-day weather-aware schedule.

        Day 0 is actionable (tonight's charge window).
        Days 1-2 are advisory (shown in dashboard, not executed).
        """
        day_index: int              # 0=today/tonight, 1=tomorrow, 2=day_after
        date: date                  # Calendar date for this day
        solar_forecast_kwh: float   # Expected solar production in kWh
        consumption_forecast_kwh: float  # Expected consumption in kWh
        net_energy_kwh: float       # solar - consumption (positive = surplus)
        confidence: float           # 1.0, 0.8, or 0.6
        charge_target_kwh: float    # Grid charge energy needed for this day
        slots: list[ChargeSlot]     # Charge slots (empty for advisory days)
        advisory: bool              # True for Day 1/2
    ```

    **Step 2: Create backend/weather_scheduler.py** (per MDS-02, MDS-03, MDS-07)

    Create WeatherScheduler class that wraps Scheduler. Key design:

    - Constructor takes: `scheduler: Scheduler`, `evcc_client`, `weather_client: OpenMeteoClient | None`, `consumption_forecaster` (with predict_hourly), `sys_config: SystemConfig`, `orch_config: OrchestratorConfig`, `tariff_engine`
    - Attributes: `active_schedule: ChargeSchedule | None = None`, `schedule_stale: bool = False`, `active_day_plans: list[DayPlan] | None = None`
    - Store `_last_solar_daily_kwh: list[float] | None = None` for deviation checks (used in Plan 02)

    `_DAY_CONFIDENCE = [1.0, 0.8, 0.6]` module constant (per MDS-03)

    Core `async def compute_schedule(self, writer=None) -> ChargeSchedule:` method:

    1. Fetch multi-day solar via `get_solar_forecast(self._evcc_client, self._weather_client)` -- returns SolarForecastMultiDay with `daily_energy_wh` in Wh. Convert to kWh: `[wh / 1000.0 for wh in solar.daily_energy_wh]`
    2. Fetch consumption via `self._consumption_forecaster.predict_hourly(72)` if available, else use a fallback of 20 kWh/day. Sum 24h chunks from `hourly_kwh` to get daily consumption kWh list.
    3. Determine `is_winter` by checking `datetime.now().month in self._sys_config.winter_months`
    4. Call `_compute_adjusted_charge(solar_daily_kwh, consumption_daily_kwh, total_capacity_kwh, is_winter)` to get `tonight_charge_kwh` and per-day deficits
    5. Build ChargeSchedule with slots using the same tariff window selection pattern as Scheduler (get cheapest slots from tariff_engine, split charge proportionally between huawei/victron by capacity ratio). Use `grid_charge_power_w=5000` for huawei, `3000` for victron.
    6. Compute `target_soc_pct` for each battery: `(charge_kwh_for_battery / battery_capacity_kwh) * 100`, clamped to SystemConfig min/max.
    7. Build 3 DayPlan objects. Day 0 gets the actual ChargeSlots, Days 1-2 get empty slots and advisory=True.
    8. Set `self.active_schedule`, `self.schedule_stale = False`, `self.active_day_plans`
    9. Store `self._last_solar_daily_kwh = solar_daily_kwh` for deviation checks
    10. If writer provided, call `writer.write_charge_schedule(schedule)` (fire-and-forget with try/except)

    `_compute_adjusted_charge` function (module-level or method):
    - For each day d in [0, 1, 2]: `effective_solar = solar_daily_kwh[d] * _DAY_CONFIDENCE[d]`, `deficit = max(0, consumption_daily_kwh[d] - effective_solar)`
    - `tonight = deficit[0] + deficit[1] * 0.5 + deficit[2] * 0.2`
    - Headroom ceiling (per MDS-07): summer headroom=0.15, winter headroom=0.05. `max_charge = total_capacity_kwh * (1.0 - headroom)`. `tonight = min(tonight, max_charge)`
    - Winter floor: `winter_floor = total_capacity_kwh * 0.30`, `tonight = max(tonight, winter_floor)` when is_winter
    - Return tonight, list of deficits, list of DayPlan-ready data

    Build `OptimizationReasoning` with descriptive text including solar/consumption/charge values.

    IMPORTANT: Do NOT delegate to `self._scheduler.compute_schedule()` to avoid double-counting solar discount (see Pitfall 2 in RESEARCH.md). Build slots directly in WeatherScheduler.
