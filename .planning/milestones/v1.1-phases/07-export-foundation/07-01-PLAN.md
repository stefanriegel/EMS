    Create `backend/export_advisor.py` with:

    1. `ExportDecision` — a string enum with STORE and EXPORT values (from enum import Enum).

    2. `ExportAdvice` — a dataclass with fields: decision (ExportDecision), reasoning (str), feed_in_rate (float), import_rate (float), forecast_demand_kwh (float), battery_soc_pct (float).

    3. `ExportAdvisor` class with constructor accepting:
       - tariff_engine: CompositeTariffEngine
       - forecaster: ConsumptionForecaster | None
       - sys_config: SystemConfig

    4. `advise(self, combined_soc_pct: float, huawei_soc_pct: float, victron_soc_pct: float, now: datetime) -> ExportAdvice` method:
       - If combined_soc_pct < 90.0: return STORE immediately (batteries not full enough)
       - If forecaster is None: return STORE with "forecaster unavailable, defaulting to STORE"
       - Call forecaster.query_consumption_history() — but since this is sync context called from async coordinator, use a cached result pattern: store last ConsumptionForecast as `_cached_forecast` and update it via a separate `async update_forecast()` method
       - Actually, simplify: make advise() synchronous, use `_cached_forecast` that is updated by `async refresh_forecast()`. If `_cached_forecast is None` or `_cached_forecast.fallback_used is True`: return STORE
       - Compute forward reserve using `_compute_forward_reserve_kwh(now)`: look at next 6 hours of tariff schedule via `get_price_schedule()`, find hours where import rate > feed_in_rate, estimate kWh needed from battery in those hours using cached forecast's `today_expected_kwh / 24.0 * expensive_hours`
       - Compare: if available battery kWh (from SoC and known ~94 kWh total pool) minus forward_reserve > 0, advise EXPORT; otherwise STORE
       - Build reasoning string including all four values: feed-in rate, current import rate, forecast demand next 4-6h, battery SoC

    5. `async refresh_forecast(self)` method: calls `self._forecaster.query_consumption_history()` if forecaster is not None, stores result in `self._cached_forecast`. Wraps in try/except, logs warning on failure, keeps stale cache.

    6. Use `logging.getLogger("ems.export_advisor")` for all logging.

    IMPORTANT: The advise() method must NEVER suggest discharging batteries to grid. It only advises on what to do with *surplus PV* — the decision is between "keep charging batteries" (STORE) vs "let surplus flow to grid" (EXPORT). This is SCO-01.
