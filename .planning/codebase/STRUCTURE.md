## Directory Purposes

**backend/:**
- Purpose: Core async FastAPI application with hardware drivers, orchestrator, and API
- Contains: Python modules for control logic, data models, external integrations
- Key files: `main.py` (entry point), `orchestrator.py` (5 s control loop), `api.py` (HTTP routes)

**backend/drivers/:**
- Purpose: Hardware abstraction layer for Huawei and Victron systems
- Contains: Protocol implementations (Modbus TCP, MQTT) and typed data models
- Key files: `huawei_driver.py`, `victron_driver.py`, model files

**frontend/src/:**
- Purpose: React dashboard for real-time monitoring and configuration
- Contains: Components, hooks, pages, TypeScript types
- Key files: `App.tsx` (root), components/* (UI cards), hooks/* (state & WebSocket)

**tests/:**
- Purpose: Unit and integration test suite for backend
- Contains: Pytest test modules organized by subsystem
- Key files: `test_orchestrator.py` (main control logic), `test_scheduler.py` (optimization)

**ha-addon/:**
- Purpose: Home Assistant add-on packaging and metadata
- Contains: Add-on config schema, entrypoint script, translation files
- Key files: `config.yaml` (user-facing options), `run.sh` (startup)

**grafana/:**
- Purpose: Pre-configured Grafana dashboards for time-series visualization
- Contains: Dashboard JSON files and InfluxDB datasource provisioning
- Key files: Dashboard definitions (auto-loaded on docker-compose startup)

## Key File Locations

**Entry Points:**
- `backend/main.py`: FastAPI app factory and lifespan context manager; production entry point for uvicorn
- `frontend/src/main.tsx`: React DOM root; mounts App component into `#root`
- `ha-addon/run.sh`: Home Assistant add-on entrypoint; exports config options to env vars

**Configuration:**
- `backend/config.py`: Dataclass configs read from environment variables (HuaweiConfig, VictronConfig, SystemConfig, etc.)
- `backend/setup_config.py`: Wizard config persistence (loads/saves `ems_config.json`)
- `pyproject.toml`: Python version, dependencies, pytest config

**Core Logic:**
- `backend/orchestrator.py`: Main 5 s control loop; computes SoC, setpoints, state transitions
- `backend/unified_model.py`: UnifiedPoolState and ControlState definitions
- `backend/scheduler.py`: Nightly charge schedule computation (starts at `SCHEDULER_RUN_HOUR`)

**API:**
- `backend/api.py`: FastAPI routes (`GET /api/state`, `POST /api/config`, `/api/tariff/*`, `/api/optimization/*`)
- `backend/setup_api.py`: Setup wizard routes (`/setup/status`, `/setup/save-config`)
- `backend/auth.py`: JWT authentication middleware and routes

**Drivers:**
- `backend/drivers/huawei_driver.py`: Modbus TCP async driver for Huawei SUN2000/LUNA2000
- `backend/drivers/victron_driver.py`: MQTT async driver for Victron Multiplus II
- Model files: `huawei_models.py`, `victron_models.py` (dataclass contracts)

**Data Persistence:**
- `backend/influx_writer.py`: Async writes to InfluxDB (metrics, schedules)
- `backend/influx_reader.py`: Async reads from InfluxDB (consumption history, forecasts)
- `backend/setup_config.py`: Persistent JSON config file path

**External Integrations:**
- `backend/evcc_client.py`: HTTP client for EVCC API (EV charging state)
- `backend/evcc_mqtt_driver.py`: MQTT driver for EVCC load monitoring
- `backend/ha_rest_client.py`: Home Assistant REST API (sensor reads, live tariff)
- `backend/ha_mqtt_client.py`: Home Assistant MQTT broker connection
- `backend/notifier.py`: Telegram alert dispatch

**Testing:**
- `tests/test_orchestrator.py`: Core control loop tests (SoC balancing, setpoints, state machine)
- `tests/test_scheduler.py`: Charge schedule computation tests
- `tests/drivers/test_huawei_driver.py`: Huawei driver unit tests
- `tests/drivers/test_victron_driver.py`: Victron driver unit tests

**Frontend:**
- `frontend/src/components/PoolOverview.tsx`: Main dashboard (SoC, power flow, warnings)
- `frontend/src/components/EnergyFlowCard.tsx`: Power flow diagram
- `frontend/src/hooks/useEmsSocket.ts`: WebSocket manager with exponential backoff reconnect
- `frontend/src/hooks/useEmsState.ts`: HTTP polling fallback

## Naming Conventions

**Files:**
- Python files: `snake_case.py` (e.g., `huawei_driver.py`, `influx_writer.py`)
- TypeScript files: `PascalCase.tsx` for components, `camelCase.ts` for utilities (e.g., `PoolOverview.tsx`, `useEmsSocket.ts`)
- Test files: `test_<module>.py` prefix (e.g., `test_orchestrator.py`)
- Config files: `<service>.yaml` or `<service>.json` (e.g., `config.yaml`, `ems_config.json`)

**Directories:**
- Python packages: `snake_case/` (e.g., `backend/`, `drivers/`)
- React directories: Plural for collections, PascalCase for single-responsibility dirs (e.g., `components/`, `hooks/`, `pages/`)

**Python Classes & Dataclasses:**
- Pattern: `PascalCase` (e.g., `HuaweiDriver`, `VictronSystemData`, `UnifiedPoolState`, `ControlState`)
- Dataclass models: Suffix with "Data" or "Config" (e.g., `HuaweiBatteryData`, `SystemConfig`, `ChargeSchedule`)

**Python Functions:**
- Pattern: `snake_case` (e.g., `read_battery()`, `write_ac_power_setpoint()`, `compute_schedule()`)
- Private functions: Leading underscore (e.g., `_huawei_sentinel()`, `_require_env()`)

**Python Variables:**
- Constants: `UPPER_CASE` (e.g., `_HUAWEI_KWH`, `_BATTERY_REGISTERS`)
- Public: `snake_case`
- Private/Internal: Leading underscore (e.g., `_start_time`)

**React Components:**
- Pattern: `PascalCase` for component names (e.g., `PoolOverview`, `EnergyFlowCard`)
- Filenames: Match component name (e.g., `PoolOverview.tsx`)

**TypeScript Types & Interfaces:**
- Pattern: `PascalCase` (e.g., `PoolState`, `DevicesPayload`, `WsPayload`, `EmsSocketState`)

**Environment Variables:**
- Pattern: `UPPER_CASE_WITH_UNDERSCORES` (e.g., `HUAWEI_HOST`, `INFLUXDB_URL`, `LOG_LEVEL`)
- Secrets: No special prefix; stored in `.env` (git-ignored)

## Where to Add New Code

**New Feature (e.g., new tariff provider):**
- Primary code: `backend/tariff.py` (add provider class), `backend/tariff_models.py` (add config dataclass)
- Tests: `tests/test_tariff_api.py` or new `tests/test_<feature>.py`
- Config: Add env var pattern to `backend/config.py` `TariffConfig.from_env()`

**New API Endpoint:**
- Implementation: `backend/api.py` (add route to `api_router`)
- Response model: `backend/api.py` (Pydantic model) or a new module if large
- Dependency: Use `get_orchestrator()`, `get_tariff_engine()`, `get_scheduler()` from DI
- Tests: `tests/test_<endpoint>.py`

**New Hardware Driver:**
- Driver class: `backend/drivers/<device>_driver.py` (implement `async connect()`, `async close()`, read/write methods)
- Data model: `backend/drivers/<device>_models.py` (dataclass contracts)
- Sentinel: Add to `orchestrator.py` if Orchestrator needs offline handling
- Tests: `tests/drivers/test_<device>_driver.py`

**New Frontend Component:**
- Component: `frontend/src/components/<ComponentName>.tsx`
- Hook (if stateful): `frontend/src/hooks/use<ComponentName>.ts`
- Page (if routable): `frontend/src/pages/<PageName>.tsx`
- Types: Add to `frontend/src/types.ts` (or inline if component-local)

**New Utilities/Helpers:**
- Shared backend helpers: `backend/<utility>.py` (e.g., `backend/validators.py`)
- Shared frontend utilities: `frontend/src/utils/<utility>.ts`

**Configuration:**
- Driver configs: `backend/config.py` (add dataclass, implement `from_env()`)
- Feature flags: `backend/config.py` (conditional instantiation in `main.py` lifespan)

## Special Directories

**frontend/dist/:**
- Purpose: Built SPA artifacts (HTML, JS, CSS)
- Generated: By `npm run build`
- Committed: No (git-ignored)
- Served: By FastAPI `StaticFiles` mount at `/` after all API routes

**frontend/test-results/ & .artifacts/:**
- Purpose: Playwright E2E test results and browser session artifacts
- Generated: By `npm run test:playwright`
- Committed: No (git-ignored)
- Contains: Screenshots, traces for debugging failed tests

**backend/__pycache__/ & .pytest_cache/:**
- Purpose: Python bytecode and pytest metadata caches
- Generated: Automatically by Python and pytest
- Committed: No (git-ignored)

**ems.egg-info/:**
- Purpose: Package metadata (generated by `pip install -e .`)
- Committed: No (git-ignored)

**ha-addon/dist/:**
- Purpose: Built Home Assistant add-on tar.gz archive
- Generated: By build script
- Committed: No (git-ignored)

**uv.lock:**
- Purpose: Deterministic lock file for `uv` package manager
- Committed: Yes (tracked in git)
- Manually managed: No (auto-generated by uv)
