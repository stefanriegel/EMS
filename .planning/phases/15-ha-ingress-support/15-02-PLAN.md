  <action>
1. In frontend/src/App.tsx, replace the hardcoded WS_URL (per INGR-04):

   Replace:
   ```typescript
   const WS_URL = `ws://${location.host}/api/ws/state`;
   ```
   With a function that constructs the URL dynamically from window.location:
   ```typescript
   function buildWsUrl(): string {
     const proto = location.protocol === "https:" ? "wss:" : "ws:";
     const wsUrl = new URL("./api/ws/state", location.href);
     wsUrl.protocol = proto;
     return wsUrl.href;
   }
   const WS_URL = buildWsUrl();
   ```
   This works under both direct access (`ws://host:8000/api/ws/state`) and Ingress (`wss://ha.local/api/hassio_ingress/{token}/api/ws/state`) because `new URL("./api/ws/state", location.href)` resolves the relative path against the current page URL, which includes the Ingress prefix when loaded via Ingress.

2. In App.tsx, change the auth check fetch from absolute to relative:
   Replace: `fetch("/api/state")`
   With: `fetch("./api/state")`

3. In frontend/src/pages/Login.tsx, change the login fetch:
   Replace: `fetch("/api/auth/login", {`
   With: `fetch("./api/auth/login", {`

4. In frontend/src/hooks/useEmsState.ts, change both fetch calls:
   Replace: `fetch("/api/state", { signal: ctrl.signal })`
   With: `fetch("./api/state", { signal: ctrl.signal })`
   Replace: `fetch("/api/devices", { signal: ctrl.signal })`
   With: `fetch("./api/devices", { signal: ctrl.signal })`

5. In frontend/src/hooks/useDecisions.ts, change the fetch call:
