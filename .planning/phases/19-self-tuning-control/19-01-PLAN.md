    **Dataclasses:**
    - `TuningParams`: huawei_deadband_w (int, default 300), victron_deadband_w (int, default 150), ramp_rate_w (int, default 2000), huawei_min_soc_profile (list[dict] | None), victron_min_soc_profile (list[dict] | None)

    **SelfTuner class:**
    - `__init__(self, state_path="/config/ems_models/tuning_state.json")` — loads persisted state or creates default. Sets `self._coordinator = None`.
    - `set_coordinator(self, coordinator) -> None` — stores a reference to the live Coordinator instance. Called once from main.py lifespan. This is the injection point that enables _apply_params() to push computed values to the coordinator's runtime fields. Per RESEARCH.md Pattern 3.
    - `record_cycle(self, pool_status: str, grid_power_w: float)` — ZERO I/O, pure in-memory. Count transitions (prev vs current pool_status). Count grid spikes (grid_power_w > 500 AND transition just happened). Roll over hourly stats after 720 cycles. Keep max 168 hourly entries (7 days). Per research Pattern 1.
    - `mark_ha_override(self, param_name: str)` — called when HA command sets a parameter; records in ha_overrides dict so nightly_tune skips it
    - `async nightly_tune(self, forecaster=None)` — the main nightly computation:
      1. Check activation gate via _check_activation_gate(forecaster). If fails, log and return.
      2. Compute 7-day average oscillation rate from _hourly_stats.
      3. Check rollback: if _check_rollback(avg_osc_rate) is True, restore previous_params, **call _apply_params() to revert coordinator fields**, save, return.
      4. Compute recommended dead-band: if avg_osc_rate > 6 trans/hour, recommend increasing by 5% of base; if < 2, recommend decreasing by 5% of base. Skip if HA-overridden.
      5. Compute recommended ramp rate: count avg daily grid spikes from _hourly_stats. If > 3/day, increase by 5% of base; if 0 for all 7 days, decrease by 5%. Skip if HA-overridden.
      6. Compute recommended min-SoC profile: call forecaster.predict_hourly() for tomorrow. Group into 4-hour blocks (0-4, 4-8, 8-12, 12-16, 16-20, 20-24). For blocks with above-average consumption, set min-SoC to 20%; for below-average, set to 10%. Output as list of MinSocWindow-compatible dicts. Skip if HA-overridden.
      7. Apply _bounded_adjust() to each numeric parameter (10% of base cap + clamp ranges).
      8. If mode == "shadow": append recommendation to shadow_log, increment shadow_days, auto-promote to "live" if shadow_days >= 14.
      9. If mode == "live": update current_params, store previous_params for rollback. **Call _apply_params() to push new values to coordinator runtime fields.**
      10. Update last_oscillation_rate. Clear ha_overrides. Save state.
    - `_apply_params(self) -> None` — **pushes current_params to the coordinator's live runtime fields.** Per RESEARCH.md Pattern 3. Guard: if `self._coordinator is None` or `self._state.mode != "live"`, return immediately (no-op). Otherwise:

      ```python
      self._coordinator._victron_deadband_w = self._state.current_params["victron_deadband_w"]
      self._coordinator._huawei_ramp_w_per_cycle = self._state.current_params["ramp_rate_w"]
      self._coordinator._victron_ramp_w_per_cycle = self._state.current_params["ramp_rate_w"]
      ```
      For min-SoC profile propagation, convert the stored list[dict] to list[MinSocWindow] and set on `self._coordinator._sys_config.huawei_min_soc_profile` and `victron_min_soc_profile` if profiles are not None. Import MinSocWindow from backend.config. Log at INFO level which parameters were applied and their new values.
    - `_check_activation_gate(self, forecaster) -> bool` — returns False if forecaster is None, MAPE >= 25.0, or days_of_history < 60
    - `_check_rollback(self, current_osc_rate: float) -> bool` — returns True if rate increased >20% vs last_oscillation_rate
    - `_bounded_adjust(self, param_name: str, current: float, base: float, recommended: float) -> float` — 10% of BASE cap, then clamp to _CLAMP_RANGES
    - `_save_state(self)` / `_load_state(self) -> TuningState` — JSON persistence with fire-and-forget error handling on save
    - `get_tuning_status(self) -> dict` — returns mode, shadow_days, current_params, recommended (last shadow entry), last_adjustment timestamp, activation_gate status
