**Part A: weather_client.py — add get_temperature_forecast()**

Add a new async method to `OpenMeteoClient`:

```python
async def get_temperature_forecast(self, hours: int = 72) -> list[float] | None:
    """Fetch hourly temperature forecast from Open-Meteo.

    Returns list of temperature_2m values in Celsius, or None on error.
    """
    params = {
        "latitude": self._config.latitude,
        "longitude": self._config.longitude,
        "hourly": "temperature_2m",
        "forecast_days": (hours + 23) // 24,
        "timezone": "UTC",
    }
    try:
        async with httpx.AsyncClient(timeout=self._config.timeout_s) as client:
            resp = await client.get(self._base_url, params=params)
            resp.raise_for_status()
            data = resp.json()
        temps = data["hourly"]["temperature_2m"][:hours]
        return temps
    except (httpx.HTTPError, KeyError, ValueError, TypeError) as exc:
        logger.warning("open-meteo get_temperature_forecast failed: %s", exc)
        return None
```

**Part B: consumption_forecaster.py — full ML upgrade**

1. **Add `_build_lag_features` helper** (module-level function after `_compute_ewm`):
   - Takes `timestamps: list[datetime]` and `consumption_map: dict[datetime, float]`
   - Returns `tuple[list[float], list[float]]` for lag_24h and lag_168h
   - Uses `float("nan")` when history is unavailable (per user decision: HistGBR handles NaN natively)

2. **Add `_compute_recency_weights` helper** (module-level function):
   - Takes `timestamps: list[datetime]` and `half_life_days: float = 30.0`
   - Returns `numpy.ndarray` of exponential decay weights
   - Formula: `weights = exp(-ln(2) / (half_life_days * 24) * age_hours)`

3. **Upgrade `_build_features`** signature to accept `consumption_map: dict[datetime, float] | None = None`:
   - Add 3 new feature columns: `is_weekend` (float, 1.0 if weekday >= 5), `lag_24h`, `lag_168h`
   - When `consumption_map` is None, use `float("nan")` for lag features
   - New feature order: `[outdoor_temp_c, ewm_temp_3d, day_of_week, hour_of_day, month, is_weekend, lag_24h, lag_168h]`

4. **Update `FEATURE_NAMES` constant** (add at module level):
   ```python
   FEATURE_NAMES: list[str] = [
       "outdoor_temp_c", "ewm_temp_3d", "day_of_week", "hour_of_day",
       "month", "is_weekend", "lag_24h", "lag_168h",
   ]
   ```

5. **Upgrade `train()` method**:
   - Replace `GradientBoostingRegressor` import with `HistGradientBoostingRegressor`
   - Add `from sklearn.model_selection import TimeSeriesSplit, cross_val_score` import

   - Pass `consumption_map` to `_build_features()` calls
   - Compute `sample_weight = _compute_recency_weights(timestamps)` and pass to `.fit()` via `sample_weight=weights`
   - For each model (heat_pump, dhw, base_load):
     - Use `HistGradientBoostingRegressor(max_iter=100, max_depth=3, random_state=42, early_stopping=True, n_iter_no_change=10, validation_fraction=0.1)`
     - Before final fit, run CV in same thread: wrap `cross_val_score(model, X, y, cv=TimeSeriesSplit(n_splits=5), scoring="neg_mean_absolute_percentage_error", fit_params={"sample_weight": weights})` inside `anyio.to_thread.run_sync`
     - Log CV scores: `logger.info("CV scores (neg_MAPE): mean=%.1f%% std=%.1f%%", -scores.mean(), scores.std())`
     - Then fit on ALL data: `model.fit(X, y, sample_weight=weights)` (also in executor)
   - Update `ModelMetadata.feature_names` to use `FEATURE_NAMES` constant
   - Wrap entire CV + fit per model in a single `anyio.to_thread.run_sync` call (per research pitfall 5)
