<objective>
Add MAPE (Mean Absolute Percentage Error) tracking to the ConsumptionForecaster and expose ML model status via a new `/api/ml/status` REST endpoint. MAPE is computed daily during the nightly retrain cycle by comparing yesterday's hourly predictions vs actual hourly consumption from HA statistics.

Purpose: The system needs to know how accurate its predictions are. MAPE tracking gates Phase 19 self-tuning (activation requires MAPE < 25%). The API endpoint makes model health visible to the user.
Output: MAPE computation + storage in `consumption_forecaster.py`, `/api/ml/status` endpoint in `api.py`, wiring in `main.py`, tests.
</objective>
