        enabled: bool = True

        baselines_path: str = "/config/ems_models/anomaly_baselines.json"
        consumption_threshold_sigma: float = 3.0
        soc_rate_threshold_sigma: float = 3.0
        efficiency_threshold_pct: float = 85.0
        minimum_consumption_hours: int = 168  # 7 days hourly
        minimum_battery_days: int = 14
        comm_loss_window_s: float = 3600.0  # 1 hour lookback
        comm_loss_min_windows: int = 3
        comm_loss_gap_s: float = 30.0  # gap between failure windows
        warning_cooldown_s: float = 3600.0  # 1 hour
        alert_cooldown_s: float = 14400.0  # 4 hours
        max_events: int = 500
        max_event_age_days: int = 90
        isolation_forest_contamination: float = 0.05
        isolation_forest_n_estimators: int = 100

        @classmethod
        def from_env(cls) -> "AnomalyDetectorConfig":
            model_dir = os.environ.get("EMS_MODEL_DIR", "/config/ems_models")
            enabled = os.environ.get("EMS_ANOMALY_ENABLED", "true").lower() == "true"
            return cls(
                enabled=enabled,
                model_dir=model_dir,
                events_path=f"{model_dir}/anomaly_events.json",
                baselines_path=f"{model_dir}/anomaly_baselines.json",
            )

    **2. Create backend/anomaly_detector.py** with:

    - `AnomalyEvent` dataclass: timestamp (ISO), anomaly_type (str: "comm_loss"|"consumption_spike"|"soc_curve"|"efficiency"), severity (str: "info"|"warning"|"alert"), message (str), value (float), threshold (float), system (str | None)

    - `_EscalationTracker` class: tracks occurrences per anomaly_type in 24h sliding window. `record(anomaly_type, now_mono) -> severity` — prune events older than 24h, append, return "alert" if count >= 3, "warning" if count >= 1

    - `_CooldownTracker` class: per-type cooldown using time.monotonic(). `can_fire(anomaly_type, severity, now_mono) -> bool` — uses warning_cooldown_s for warnings, alert_cooldown_s for alerts. `record(anomaly_type, severity, now_mono)` updates last-fire time.

    - `HourlyBaseline` dataclass: mean, std, count. `update(value, alpha=0.1)` — exponential moving average as shown in RESEARCH.md.

    - `SocBandBaseline` dataclass: per-band (0-20, 20-50, 50-80, 80-100) tracking of charge/discharge rate (delta_soc/delta_time). Same EMA update pattern. Store as dict keyed by band string and direction ("charge"/"discharge").

    - `AnomalyDetector` class:
      - `__init__(self, cfg: AnomalyDetectorConfig, model_store: ModelStore | None = None)` — load events from events_path, load baselines from baselines_path, init escalation/cooldown trackers, init hourly baselines (24 slots), init SoC band baselines (4 bands x 2 directions), init charge/discharge kWh accumulators per system, init comm failure history per system.
      - `check_cycle(self, h_snap: ControllerSnapshot | None, v_snap: ControllerSnapshot | None) -> list[AnomalyEvent]` — MUST be pure float comparisons. Calls _check_comm_loss, _check_consumption, _check_soc_rate, _check_efficiency. Returns list of events. Updates internal tracking state. Saves events to disk after appending (fire-and-forget try/except).
      - `_check_comm_loss(h_snap, v_snap) -> list[AnomalyEvent]` — per RESEARCH.md pattern: track failure timestamps per system, count distinct windows (gaps > 30s = new window), trigger if >= 3 windows in last hour.
      - `_check_consumption(h_snap, v_snap) -> list[AnomalyEvent]` — derive system activity from snapshots, update hourly baseline for current hour, check if value > mean + threshold_sigma * std. Only after minimum_consumption_hours updates.
      - `_check_soc_rate(h_snap, v_snap) -> list[AnomalyEvent]` — track SoC rate per band when power != 0, accumulate over 5-min windows, compare against 7-day rolling baseline. Only after minimum_battery_days of data.
      - `_check_efficiency(h_snap, v_snap) -> list[AnomalyEvent]` — accumulate charge/discharge kWh per system. Every 24h (based on monotonic clock), compute efficiency = discharge/charge, flag if < 85%. Reset accumulators.
      - `_track_efficiency(name, snap, dt_s)` — as shown in RESEARCH.md.
      - `async nightly_train(self) -> None` — gather recent data from baselines, build feature matrix, train IsolationForest via `await anyio.to_thread.run_sync(model.fit, X_train)`, extract thresholds via decision_function + percentile, save model via ModelStore, save baselines to JSON.
      - `get_events(self, limit: int = 100) -> list[dict]` — return recent events as dicts.
      - `get_battery_health(self) -> dict` — return per-system efficiency (last 24h), SoC band baselines, data age.
      - `_save_events()` / `_load_events()` — JSON persistence following MAPE history pattern. Atomic write (write to tmp, rename). Trim to max_events. Prune events older than max_event_age_days.
      - `_save_baselines()` / `_load_baselines()` — JSON persistence for hourly baselines, SoC band baselines, efficiency accumulators.

    Logger: `logging.getLogger(__name__)` as `logger`.

    All escalation/cooldown checks happen inside check_cycle before appending to events list. Only events that pass cooldown get appended and persisted.
