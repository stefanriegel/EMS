  <behavior>
    ## P_target computation
    - _compute_p_target(): uses victron_snapshot.grid_power_w as primary source
    - _compute_p_target(): falls back to huawei_snapshot.master_active_power_w when victron grid_power_w is None or victron unavailable
    - _compute_p_target(): returns 0.0 when both sources unavailable

    ## SoC-based role assignment (D-01, D-02, D-08, CTRL-06, CTRL-08)
    - _assign_discharge_roles(h_soc, v_soc): higher SoC system gets PRIMARY_DISCHARGE
    - When SoC gap >= 5%: PRIMARY discharges, other gets HOLDING (not SECONDARY)
    - When SoC gap < 5%: PRIMARY discharges, other gets SECONDARY_DISCHARGE
    - Swap hysteresis: current PRIMARY keeps role unless challenger is >3% higher (D-02)
    - Both below min SoC: both get HOLDING

    ## Allocation (CTRL-02, CTRL-05)
    - _allocate(): PRIMARY gets full P_target (negative watts = discharge demand)
    - _allocate(): when SECONDARY active (gap < 5%), split proportionally by available capacity
    - _allocate(): when one system unavailable, entire P_target goes to survivor (D-10)
    - Coordinator sends ControllerCommand — never calls driver methods (CTRL-02)

    ## PV surplus routing (D-03, D-04)
    - _allocate_charge(surplus_w): fills Huawei (30kWh) first, then Victron (64kWh) (D-03)
    - When one battery at 95% SoC: all surplus routes to the other (D-04)
    - Respects charge_headroom_w from each controller's snapshot

    ## Hysteresis (CTRL-03, D-06)
    - _apply_hysteresis(): Huawei dead-band configurable, default 300W — suppress changes < dead-band
    - _apply_hysteresis(): Victron dead-band configurable, default 150W — suppress changes < dead-band
    - Hysteresis applied by coordinator BEFORE sending to controllers (D-06)
    - Dead-band values stored in coordinator config (new fields: huawei_deadband_w, victron_deadband_w)

    ## Ramp limiting (CTRL-07)
    - _apply_ramp(): limits setpoint change per cycle to configurable max ramp rate
    - Separate ramp rates for Huawei and Victron (configurable)
    - Default ramp: Huawei 2000 W/cycle, Victron 1000 W/cycle

    ## Debounce (D-16)
    - _debounce_role(): role transitions require 2 consecutive cycles proposing same role
    - Safe state transitions (HOLDING due to comms loss) bypass debounce — immediate
    - Tracks pending role and pending cycle count per controller

    ## Grid charge (D-08)
    - _check_grid_charge(): checks active charge slot from scheduler
    - When slot active: sends GRID_CHARGE role with per-battery charge watts to each controller
    - On slot exit: sends cleanup command (HOLDING with 0 watts) — Pitfall 3

    ## EVCC hold (D-07)
    - When evcc_battery_mode == "hold": coordinator sets evcc_hold=True on all commands
    - Each controller receives the hold signal and transitions independently

    ## State building
    - _build_state(): constructs CoordinatorState from both snapshots and commands
    - Includes all backward-compat fields (combined_soc_pct computed as 30/64/94 weighted average)
    - Includes new per-system fields: huawei_role, victron_role, pool_status
    - pool_status: NORMAL (both available), DEGRADED (one available), OFFLINE (neither)
    - get_state() returns the current CoordinatorState or None before first cycle

    ## Control loop
    - _run_cycle(): poll both controllers, compute P_target, assign roles, allocate, apply hysteresis, apply ramp, send commands, build state
    - start(): creates asyncio task running the control loop at loop_interval_s
    - stop(): cancels the task, waits for cleanup
    - Coordinator constructor: __init__(self, huawei_ctrl, victron_ctrl, sys_config, orch_config, writer=None, tariff_engine=None)
    - set_scheduler(scheduler), set_evcc_monitor(evcc_mqtt), set_notifier(notifier) — same interface as current Orchestrator

    ## Reconnect behavior (D-12)
    - When a controller reports consecutive_failures >= 3 for N cycles, coordinator calls controller's driver.connect() periodically (every 30s)
    - On successful reconnect, controller resumes normal polling

    ## Config update
    - sys_config property setter: updates SystemConfig (thread-safe via GIL, same as current Orchestrator pattern)
