<objective>
Wire ModelStore into the application, make ConsumptionForecaster training non-blocking via executor, and set OMP_NUM_THREADS in Docker.

Purpose: Without executor wrapping, model.fit() blocks the async event loop for 15-60s on aarch64, freezing the 5s control loop. Without OMP_NUM_THREADS, sklearn oversubscribes CPU threads in containers. Without ModelStore wiring, trained models are lost on restart.

Output: Modified `consumption_forecaster.py` with executor + ModelStore, updated `main.py` lifespan, `Dockerfile` and `run.sh` with thread limits, new test for executor behavior.
</objective>
