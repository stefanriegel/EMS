  <behavior>
    HCTL-01 — Startup activation:
    - test_activate_clamps_power_first: activate() calls write_max_charge_power(0) and write_max_discharge_power(0) BEFORE write_battery_mode(TIME_OF_USE_LUNA2000)
    - test_activate_writes_tou_mode: activate() calls write_battery_mode(StorageWorkingModesC.TIME_OF_USE_LUNA2000)
    - test_activate_transitions_to_active: after activate() completes, state is ModeState.ACTIVE

    HCTL-02 — Shutdown restore:
    - test_restore_writes_self_consumption: restore() calls write_battery_mode(StorageWorkingModesC.MAXIMISE_SELF_CONSUMPTION)
    - test_restore_idempotent: restore() does not raise if driver raises (logs WARNING, swallows)
    - test_crash_recovery_skips_clamping: activate() with current_working_mode=5 (already TOU) skips clamping, goes straight to ACTIVE

    HCTL-03 — Health check:
    - test_health_check_reapplies_on_revert: check_health(current_working_mode=2) when state is ACTIVE triggers re-apply (clamp + switch)
    - test_health_check_noop_when_correct: check_health(current_working_mode=5) when ACTIVE does nothing
    - test_health_check_cooldown: after re-apply, subsequent check_health calls within cooldown (30s) are skipped
    - test_health_check_respects_interval: check_health only runs when health_check_interval_s has elapsed

    HCTL-04 — Transition safety:
    - test_is_transitioning_during_activate: is_transitioning is True during CLAMPING and SWITCHING states
    - test_is_transitioning_false_when_active: is_transitioning is False in ACTIVE state
    - test_activate_waits_settle: activate() includes asyncio.sleep(settle_delay_s) between clamp and mode switch, and between switch and ACTIVE
  </behavior>
